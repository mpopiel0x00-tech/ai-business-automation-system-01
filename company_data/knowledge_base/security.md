FlowForge is designed with security as a core principle. The platform includes multiple layers of protection to ensure that customer data remains secure and reliable.

---

## Data Encryption

All data transmitted between users and FlowForge servers is encrypted using TLS encryption.

Sensitive data stored within the platform is encrypted at rest using industry-standard encryption protocols.

These protections ensure that customer information remains secure during both transmission and storage.

---

## Single Sign-On (SSO)

Enterprise customers can enable Single Sign-On (SSO) to allow team members to log in using their organization's identity provider.

Supported identity providers include:

- Google Workspace
- Okta
- Microsoft Azure Active Directory

SSO reduces password management risks and simplifies authentication for large organizations.

---

## Audit Logs

FlowForge provides audit logging capabilities that track important events within a workspace.

### Logged Events

- user logins
- permission changes
- workflow modifications
- automation rule updates
- data exports

Audit logs help organizations monitor activity and maintain operational transparency.

---

## Compliance

FlowForge follows modern cloud security standards and operational practices.

Compliance practices include:

- secure infrastructure management
- controlled access to production systems
- continuous monitoring
- data protection policies

Enterprise customers can request additional compliance documentation when required.

---

## Infrastructure Security

FlowForge infrastructure is hosted in secure cloud environments with the following protections:

- network isolation
- firewall protections
- regular security monitoring
- automated backups
- infrastructure redundancy

These safeguards help maintain high availability and data integrity.
